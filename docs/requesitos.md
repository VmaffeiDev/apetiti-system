📄 2. REQUISITOS
✅ Requisitos Funcionais (RF)

RF01 – O sistema deve iniciar a interação com saudação
RF02 – O sistema deve identificar se o usuário deseja almoço ou jantar
RF03 – O sistema deve sugerir uma refeição com base no contexto
RF04 – O sistema deve permitir feedback do usuário (👍 ou 👎)
RF05 – O sistema deve armazenar preferências básicas do usuário
RF06 – O sistema deve adaptar sugestões com base no histórico
RF07 – O sistema deve manter o estado da conversa (diálogo contínuo)
RF08 – O sistema deve tratar variações de entrada (ex: “almoçar”, “almoco”)

⚙️ Requisitos Não Funcionais (RNF)

RNF01 – O sistema deve responder em tempo real (< 2 segundos)
RNF02 – O sistema deve estar disponível via Telegram
RNF03 – O sistema deve ser simples e intuitivo
RNF04 – O sistema deve funcionar sem necessidade de login
RNF05 – O sistema deve manter dados em memória na Sprint 01
